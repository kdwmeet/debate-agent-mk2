# Multi-Agent Debate System (자율 토론 및 합의 도출 시스템)

## 1. 프로젝트 개요

이 프로젝트는 LangGraph를 활용하여 서로 다른 페르소나(이해관계)를 가진 여러 에이전트가 자유롭게 의견을 교환하고 최적의 결론을 도출하는 다중 에이전트(Multi-Agent) 토론 시스템입니다.

일방적인 지시가 아닌, 원가 절감 전문가, 품질 극대화 전문가, 마케팅 전문가가 신제품 기획을 두고 치열하게 논쟁합니다. 이 과정에서 중재자(CEO) 에이전트가 대화의 흐름을 제어하고, 충분한 합의점이 도출되거나 제한된 발언 횟수에 도달하면 최종 결론을 요약하고 토론을 종료합니다.

## 2. 시스템 아키텍처

본 시스템은 중재자를 중심으로 여러 에이전트가 턴을 주고받는 순환 네트워크 구조를 가집니다.

1. State Definition: 토론 주제, 대화 기록(messages), 진행된 턴 수(turn_count), 다음 발언자, 최종 요약문을 전역 상태로 관리합니다.
2. Moderator Node: 대화 맥락을 분석하여 다음에 반박이나 의견을 제시할 에이전트를 동적으로 라우팅합니다. 무한 논쟁을 방지하기 위해 최대 턴 수에 도달하면 강제로 FINISH 상태로 전환하고 회의록을 요약합니다.
3. Expert Nodes (Cost, Quality, Marketing): 각각 원가 절감, 최고급 품질, 시장 트렌드라는 명확한 목적 함수(페르소나)를 가지고 상대방의 의견에 반박하며 주장을 펼칩니다.
4. Routing Logic: 전문가 노드가 발언을 마치면 제어권은 무조건 다시 중재자에게 넘어가며, 중재자의 판단에 따라 다음 발언자로 향하는 동적 라우팅이 발생합니다.

## 3. 기술 스택

* Language: Python 3.10+
* Package Manager: uv
* LLM: OpenAI gpt-5-mini
* Data Validation: Pydantic (v2) (중재자의 라우팅 결정 및 요약 구조화)
* Orchestration: LangGraph (동적 라우팅 및 상태 공유), LangChain
* Web Framework: Streamlit

## 5. 설치 및 실행 가이드

### 5.1. 환경 변수 설정
프로젝트 루트 경로에 .env 파일을 생성하고 API 키를 입력하십시오.

OPENAI_API_KEY=sk-your-api-key-here

### 5.2. 의존성 설치 및 앱 실행
독립된 가상환경을 구성하고 애플리케이션을 구동합니다.

uv venv
uv pip install -r requirements.txt
uv run streamlit run main.py

## 6. 테스트 시나리오 및 검증 방법

* 안건 발제: "20대 대학생을 위한 5만 원 이하의 노이즈 캔슬링 무선 이어폰 기획"과 같은 구체적인 주제를 입력하고 토론을 시작합니다.
* 페르소나 충돌 관찰: 원가 전문가는 기능 축소를, 품질 전문가는 성능 유지를, 마케팅 전문가는 디자인과 타협안을 제시하며 서로 논리적으로 반박하는 과정을 확인합니다.
* 동적 라우팅 확인: 정해진 순서가 아니라, 직전 발언자의 주장에 가장 크게 반발할 만한 에이전트를 중재자가 적절히 선택하여 마이크를 넘기는지 점검합니다.
* 합의 도출 및 종료: 설정된 최대 턴 수에 도달하거나 의견이 좁혀졌을 때, 중재자가 토론을 종료(FINISH)하고 흩어진 의견들을 모아 하나의 완성된 최종 회의 요약록을 출력하는지 확인합니다.
